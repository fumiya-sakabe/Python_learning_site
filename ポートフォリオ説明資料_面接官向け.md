## 🛠️ 技術スタック

### バックエンド
- **Flask 3.0.0**
  - **選定理由：** シンプルで学習しやすく、小規模から中規模のWebアプリケーションに適している。Python初心者にも理解しやすいフレームワーク。

### 認証・セッション管理
- **Flask-Login 0.6.3**
  - **選定理由：** ユーザー認証とセッション管理を簡単に実装できる。ユーザーごとの進捗管理に必要。

### データ管理
- **SQLite（データベース）**
  - **選定理由：** 小規模アプリケーションに適しており、設定が簡単。ファイルベースで運用しやすい。
- **JSON（ユーザー進捗データ）**
  - **選定理由：** ユーザーごとの進捗データをJSON形式で保存。シンプルで拡張しやすい。

### フロントエンド
- **HTML5 / CSS3 / JavaScript**
  - **選定理由：** 標準的なWeb技術。レスポンシブデザインを実装し、モバイルでも使いやすい。

### その他
- **Markdown 3.5.1**：レッスンコンテンツをMarkdown形式で管理
- **Pygments 2.16.1**：コードシンタックスハイライト
- **Gunicorn 21.2.0**：本番環境でのWSGIサーバー

## 🤖 AIツール（Cursor）の活用

### 活用した場面
1. **データベース設計の初期案**：テーブル構造の提案を生成し、それを理解して修正
2. **CSSのレスポンシブ対応**：メディアクエリのコードを生成し、プロジェクトに合わせて調整
3. **JavaScriptのイベント処理**：DOM操作のコード例を生成し、機能に合わせてカスタマイズ
4. **エラーハンドリング**：例外処理のパターンを生成し、適切に実装

### 理解と改善
- 生成されたコードは必ず理解してから使用
- プロジェクトの要件に合わせて修正・改善
- エラーが出た場合は自分でデバッグして解決

### 学んだこと
- AIツールは学習の補助として非常に有効
- ただし、生成されたコードを理解することが重要
- 実際の開発では、要件に合わせた調整が不可欠

## 📊 開発で学んだこと・苦労した点

### 学んだこと
1. **Flaskの基本から応用まで**
   - ルーティング、テンプレート、フォーム処理
   - セッション管理、ユーザー認証
   - データベース連携（SQLite）

2. **フロントエンド開発**
   - レスポンシブデザインの実装
   - JavaScriptによる動的なUI操作
   - CSS Grid/Flexboxによるレイアウト

3. **デプロイの実践**
   - Renderでのデプロイ手順
   - 環境変数の設定
   - 本番環境でのエラーハンドリング

### 苦労した点と解決方法
1. **レスポンシブデザインの実装**
   - **問題：** モバイルでの表示が崩れる
   - **解決：** メディアクエリを適切に設定し、モバイルファーストのアプローチで実装

2. **ユーザー進捗データの管理**
   - **問題：** JSONファイルの読み書きでエラーが発生
   - **解決：** エラーハンドリングを追加し、ファイルが存在しない場合の処理を実装

3. **Markdownのレンダリング**
   - **問題：** コードブロックのシンタックスハイライトが正しく表示されない
   - **解決：** Pygmentsライブラリを使用してコードハイライトを実装
